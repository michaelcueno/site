Design

This is the login screen of the application. Users can also access the common account just in case they don't have an account. Users can also create a new account.

If the user (with an account) has enabled the use of password, then they login by entering their password. Otherwise, just by selecting the account from the previous page, users are logged in.

The new user enters the information, and then the account is created accordingly.

After that (i.e. after logging in any way), we are at the main screen of the application. The main screen tiles all the options. Options like clock, calendar, notes/to-do lists, commuting info, weather info, social/news feeds, kitchen support, music player and application settings

This is the normal look of the clock. This clock has a world clock, where you can add other cities to look up time. It has an alarm clock, a timer and a stop watch.

Each of the features of the clock can be expanded (physically on screen) by dragging the boundaries. This image shows on such scenario.

This image shows the detailed view of the music player.

This image depicts all the weather information.

This is the detailed view of the calendar. The crosses in the calendar depict that that day is marked by the user. A day in the calendar can be marked for any of the three things - 1. Memo, 2. Birthday, 3. Reminder. Select a cell in the calendar to add on these three to that day.

This view shows how you can mark a day in the calendar.

This image shows the page where you enter the information about birthday (or a memo or a reminder) that you are marking the calendar for.

This image shows the detailed image of notes/to-do lists page of the application. Using this feature, users can make notes, to-do lists and shopping lists/inventories.

This is the new note page, where you can create a new note.

This is the page where users can view to-do lists. They can go to the new to-do list from this page. They can also view the list by selecting a list.

This is an example of how the list looks when selected.

This is an example of the page to create new to-do lists. Grocery lists/inventories are similar to to-do lists. The only difference is that, you can also edit the grocery list.

This is the kitchen support page. I am not clear on how we are going to implement it. So this part of the applications is quite fuzzy.

This image shows the page where you can look at commuter information. Here you can save multiple locations, and by selecting a location, you can see the routes, and commuter info for that location. You can edit the arrival/departure times. And also the current location can be changed. Also, you can add a new location.

This image depicts the news feed page of the application. Each tab displays the news headlines. Users can select a particular tab to see the details.

This is the settings page of the application. Here the users can manage their accounts, and manage certain application features like changing the language, etc.
