Login Screen

The login screen was designed to mimic blinds over a window. It was used to give the user the impretion that they were still in the kitchen. In addition, we added the blinds to seperate user applications from public applications, parents don't normally want their kids to mess around with their Gmail, Facebook, and or Twitter accounts. So to login the user will have to enter a password. Behind the blinds is the window, were all the applications will be loaded into (as if you were looking out the kitchen window).

This is the login screen. The wooden panel highlighted above, is a button that will log into the users account when pressed (and password provided, if necessary). The buttons below the user panels are quick buttons that will launch one of the many applications provided by KIT



When the user selects one of the buttons from the quick button menu, let's say the calculator, the wooden panels will be pulled up by the above animation and the application will displayed in the window behind the blinds. A simulation depicting blinds being pulled up from any given window in the kitchen already, which will make the user feel as if they are still in kitchen.




Calculator Application


The above screen is the calculator application. It can act as a standard and/or a conversion calculator. It is seperated into four panes the number panel (1), operator panel (2), output (3) and the mode selection (4). The above picture shows that the calculator is currently selected as a standard calculator. The number two is selected.




After pushing the number two the value will be displayed in the output panel. The user may now press an operator to continue the equation.



The next value selected is two (as displayed above). The user then may press the equals button to finish the equation (as displayed above)



The equation will be calculated and displayed in the output pane (as above) 2 * 2 = 4



Now let's say the user is baking some almond cookies, but the labels on the packages of the necessary ingrediants are labeled in pounds and the user needs them in ounces. Well the user can open the calculator application select "lbs -> oz" in the mode selection pane and then input the desired amount in pounds



When the user is done inputing the amount in pounds they need only hit the equals button to have the calculator convert it into ounces and display it in the output pane.



Recipe Application



This is the recipe window. This is where all the recipes from the applications API are loaded in to. As for right now the API only has a few items loaded into it, so there aren't that many recipes to look at. This is denoted by the coloring scheme of the index on the right of the above image. Solid black indices mean that there are recipes stored under that given index, and greyed indices mean that there are none.



When you select one of the black indices, a panel will slide out and all the available recipes (under that index) will be listed in it (as seen above). When the user has found the recipe that they like, they may load it by clicking on the name.



After it loads four panes will be displayed: instructions (1) ingrediants that are already owned and in the pantry (2 top pane), ingrediants that are needed but not in the pantry (2 bottom pane), nutritional graph (3), and the recipes name (4). The instruction pane slides up and down and is sepereated by step. The ingrediant panes are self explanitory and are loaded based on what is present in the pantry application and what is needed by the recipe. The nutritional graph shows the total amount of calories, protein, fat and carbs per serving size.

User Screen



The above application is the User Screen. This is loaded when one of the user panels from the main menu is selected. This application is user specific and is linked to that users: Email, RSS feeds, Twitter, and Facebook accounts. All updates/notifications are displayed in the window, and are easily accessible.



The above picture shows how the notifications are swipable and the user can cycle through them in order of arrival. If the user would like to see more detail on the notification, they may click on it and KIT will load the website in the main window.



As the above image shows the window can be scrolled up and down as well.



Settings Screen



The above is the settings window. Here the user can change the language, add new users to KIT, change the cooking API to another services API and change from metric to US standard form of measurement. As seen from the image above the defualt language is English. All changes are saved when you switch to a new application and/or panel



If the user wants to change the language all they must do is select one of the flags from the slider to the right of change language, and all text in the application will be translated to the selected language. They are shown as flags, so that if anyone were to select a language that they didn't know, by accident, they would be easily able to find their own language again by a countries flag.



To change from US standard to the metric system, all the user has to do is select the icon that shows the UN flag (since the rest of the world uses metric), and all means of measurement will be shown in metric standard. If they want US standard all they have to do is press the US flag and it will be switched back.



Lastly if the owners of KIT want to add another user to the system, all they have to do is go to the settings click on add new user and fill in all the desired information. If the new user has a Facebook, Twitter or Gmail account they may fill in their information and KIT will associate the User Screen (from before) with their accounts.
